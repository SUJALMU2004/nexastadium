# NexaStadium AI Solution Map

| Problem Domain | How It Is Solved | Key Files |
| --- | --- | --- |
| Fan assistance | Streaming stadium assistant plus polished deterministic stadium guidance | `client/src/pages/fan/AIAssistant.jsx`, `server/app/services/ai_service.py::get_fan_assistant_response` |
| Fan navigation | AI wayfinding chat plus step-by-step stadium directions | `server/app/services/ai_service.py::get_navigation_guidance`, `client/src/pages/fan/StadiumNavigator.jsx` |
| Fan safety | Multilingual safety support packs using local safety templates and accessibility context | `client/src/pages/fan/SafetyGuide.jsx`, `server/app/services/ai_service.py::generate_safety_support_pack`, `server/app/data/multilingual_safety_templates.json` |
| Crowd management | Six realistic scenarios plus crowd risk simulation and scenario comparison | `server/app/services/scenario_service.py`, `server/app/services/simulation_service.py`, `client/src/pages/ops/CrowdSimulator.jsx` |
| Accessibility | Dedicated guide, mobility-aware navigation, safety pack accessibility notes, and structured venue features | `client/src/pages/fan/AccessibilityGuide.jsx`, `client/src/pages/fan/StadiumNavigator.jsx`, `server/app/data/stadiums.json::accessibility_features`, `server/app/data/accessibility_playbooks.json` |
| Transportation | Egress planner, route recommender, alerts, and transit/gate flow-control planner | `server/app/services/transit_service.py`, `server/app/services/simulation_service.py::generate_flow_control`, `client/src/pages/transit/EgressPlanner.jsx`, `client/src/pages/transit/RouteRecommender.jsx`, `client/src/pages/transit/TransportAlerts.jsx`, `client/src/pages/transit/FlowControl.jsx` |
| Multilingual assistance | Six-language i18n, RTL Arabic, native language switcher, and language parameters in AI calls | `client/src/i18n.js`, `client/src/components/common/LanguageSwitcher.jsx`, `client/public/locales`, `server/app/services/ai_service.py` |
| Operational intelligence | AI-shaped incident action plans using scenario, severity, affected zone, and sanitized notes | `client/src/pages/ops/IncidentManager.jsx`, `server/app/services/ai_service.py::get_ops_recommendation`, `server/app/api/routes/ai_routes.py` |
| Real-time decision support | PA announcement generator plus match-day briefing grounded in scenario and local knowledge context | `client/src/pages/ops/AnnouncementDraft.jsx`, `client/src/pages/ops/MatchDayBriefing.jsx`, `server/app/services/ai_service.py::draft_pa_announcement`, `server/app/services/ai_service.py::generate_match_day_briefing` |
| Sustainability | Deterministic carbon, waste, refill, energy, and transit adoption dashboard plus report notes | `client/src/pages/ops/SustainabilityTracker.jsx`, `client/src/pages/ops/OperationsReport.jsx`, `server/app/services/report_service.py` |
| Volunteer coordination | Scenario-driven staff task recommendations and briefing volunteer coordination notes | `client/src/pages/ops/OpsDashboard.jsx`, `client/src/pages/ops/MatchDayBriefing.jsx`, `server/app/services/scenario_service.py` |
| Local knowledge retrieval | JSON-backed RAG-style search by query, stadium, category, tags, and priority | `server/app/services/knowledge_service.py`, `server/app/api/routes/knowledge_routes.py`, `server/app/data/knowledge_base.json` |
| Operations reporting | Structured summary plus copyable/downloadable Markdown report generated client-side from backend response | `client/src/pages/ops/OperationsReport.jsx`, `server/app/services/report_service.py`, `server/app/api/routes/report_routes.py` |
| Problem alignment data | All 16 official FIFA World Cup 2026 host stadiums and six scenario records are local, structured, and cached | `server/app/data/stadiums.json`, `server/app/data/scenarios.json`, `server/app/services/stadium_service.py` |
| Security | No auth, prompt guard, rate limiting, CORS from settings, backend-only OpenRouter key, no raw provider errors | `server/app/core/prompt_guard.py`, `server/app/main.py`, `server/app/core/config.py`, `client/src/services/api.js` |
| AI provider integration | OpenRouter is the backend-only AI gateway with environment-based model selection and deterministic stadium responses | `server/app/services/ai_service.py`, `server/app/core/config.py`, `server/.env.example` |
